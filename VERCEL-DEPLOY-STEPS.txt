VERCEL PATCH APPLY STEPS - ALLIED SCHOOL MANAGEMENT HUB

This patch fixes the Vercel deploy dependency issue.

STEP 1 - Extract this patch zip
Extract the zip I gave you.

STEP 2 - Copy files into GitHub Desktop folder
Copy these files/folders into your GitHub Desktop project folder and replace existing files:

- api/[...path].ts
- package.json
- pnpm-lock.yaml
- vercel.json
- VERCEL-DEPLOY-STEPS.txt

STEP 3 - Push with GitHub Desktop
1. Open GitHub Desktop.
2. Summary:
   fix vercel deployment dependencies
3. Click Commit to main.
4. Click Push origin.

STEP 4 - Vercel
Go to Vercel. It should automatically redeploy after push.
If it does not, open the project and click Redeploy on the latest deployment.

Correct Vercel settings:

Root Directory:
./

Build Command:
pnpm install --frozen-lockfile && PORT=3000 BASE_PATH=/ NODE_ENV=production pnpm --filter @workspace/allied-school run build

Output Directory:
artifacts/allied-school/dist/public

Environment Variables:
DATABASE_URL = your Neon PostgreSQL connection string
NODE_ENV = production
BASE_PATH = /

Default demo logins:
admin / allied2024
director / director2024
principal / principal2024

Important:
The red sourcemap lines in Vercel logs are warnings. The real deployment error is usually at the very bottom of Deploy Logs.
